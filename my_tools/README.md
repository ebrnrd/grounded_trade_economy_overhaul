# generate_trade_buy.py — README

## What This Tool Does

This tool generates the `[trade_generic_buy]` section for a STALKER Anomaly
trader LTX file. Instead of manually listing every item with its buy price
multipliers, you maintain two simple input files — one defining item categories
and one defining the min/max multipliers per category for a specific trader —
and the tool expands them into the full item list automatically.

This means you never have to touch individual item entries. To change how much
Sidorovich pays for common weapons, you change one line in the multipliers file
and re-run the tool.

---

## Files You Need

Place all files in the same folder before running.

### 1. `generate_trade_buy.py`
The tool itself. Do not edit this file.

### 2. `trade_categories.ltx`
Defines your item categories. Each section header is a category name, and each
line inside it is an item name exactly as it appears in the game's LTX files.

Example:
```
[wpn_low_tier]
wpn_pm
wpn_fort
wpn_ak74u
wpn_bm16
wpn_toz34

[wpn_high_tier]
wpn_svd
wpn_vintorez
wpn_val

[food_basic]
bread
conserva
meat_boar
meat_dog
```

Rules:
- Section names can be anything you want — they are your labels, not game values.
- Item names must match exactly what the game uses (copy from the game's LTX files).
- Lines starting with `;` are comments and are ignored.
- Blank lines are ignored.

### 3. `trader_categories_multipliers.ltx`
Defines the buy price range for each category for a specific trader. The section
name identifies the trader (for your reference only — the tool reads whichever
section is present). Each line maps a category name to a `max, min` multiplier pair.

Example:
```
[sidorovich]
wpn_low_tier   = 0.55, 0.35
wpn_high_tier  = 0.10, 0.05
food_basic     = 0.65, 0.45
ammo_common    = 0.60, 0.40
```

Rules:
- The first number is the MAXIMUM multiplier (best price the trader can offer).
- The second number is the MINIMUM multiplier (worst price the trader can offer).
- The engine picks a random value between min and max each transaction.
- If you accidentally write them in the wrong order, the tool will swap them
  and warn you.
- Categories listed here but not found in `trade_categories.ltx` will be skipped
  with a warning.
- Categories defined in `trade_categories.ltx` but not listed here will be
  skipped with a warning.

## How the Multipliers Work in Game

The final price the player receives when selling an item is:

```
final_price = base_price × global_discount × random(min, max) × condition_factor
```

The `global_discount` is set in `trade_discounts.ltx` for each trader.
The `random(min, max)` is what this tool controls.

This means if Sidorovich has a global discount of `0.35` and you set
`wpn_low_tier = 0.55, 0.35`, the effective range for a low tier weapon is:

```
base_price × 0.35 × 0.35  =  ~12% of base  (bad roll)
base_price × 0.35 × 0.55  =  ~19% of base  (good roll)
```

Set your multipliers with this stacking in mind. Higher global discount =
lower per-item multipliers needed to achieve the same result.

## How To Run It (Windows)

### First time only — Install Python
1. Go to https://www.python.org/downloads/
2. Download and run the installer.
3. On the first screen, check the box that says **"Add Python to PATH"** before
   clicking Install. This is easy to miss and required.

### Every time you want to generate output
1. Put `generate_trade_buy.py`, `trade_categories.ltx`, and
   `trader_categories_multipliers.ltx` in the same folder.
2. Open that folder in Windows Explorer.
3. Click the address bar at the top of the Explorer window, type `cmd`,
   press Enter. A Command Prompt window opens pointing at your folder.
4. Type the following and press Enter:

```
python generate_trade_buy.py
```

5. The tool creates `trade_generic_buy_output.ltx` in the same folder.
6. Open it with any text editor (Notepad, Notepad++, VS Code, etc.).
7. Copy the contents into your trader's LTX file, replacing the existing
   `[trade_generic_buy]` section.

### Custom file names
If your files have different names, use the optional arguments:

```
python generate_trade_buy.py --categories my_categories.ltx --multipliers my_multipliers.ltx --output sidorovich_output.ltx
```

### Getting help
```
python generate_trade_buy.py --help
```

## Understanding the Output

The output file looks like this:

```
; Auto-generated by generate_trade_buy.py
; Do not edit manually — edit trade_categories.ltx and
; trader_categories_multipliers.ltx then re-run the generator.

[trade_generic_buy]:trasher
; --- wpn_low_tier (max=0.55, min=0.35)
wpn_pm                                   = 0.55, 0.35
wpn_fort                                 = 0.55, 0.35
wpn_ak74u                                = 0.55, 0.35
...

; --- food_basic (max=0.65, min=0.45)
bread                                    = 0.65, 0.45
conserva                                 = 0.65, 0.45
...
```

Items are grouped by category with a comment header showing the multiplier range.
Do not edit this file directly — edit the two input files and re-run the tool.

## Warnings and Errors

The tool prints warnings for common mistakes:

| Warning | Meaning | Fix |
|---|---|---|
| `category 'X' defined in multipliers but not found in categories file` | You referenced a category name that doesn't exist | Check spelling in both files |
| `category 'X' defined in categories file but has no multiplier` | A category exists but wasn't assigned a price range | Add it to the multipliers file or remove it from categories |
| `expected 'max, min' for 'X'` | A line in the multipliers file isn't formatted correctly | Make sure the format is `category_name = 0.5, 0.3` |
| `min > max for 'X' — swapping` | You wrote the numbers in the wrong order | The tool fixes it automatically, but correct your file |

Warnings do not stop the tool — output is still generated for all valid entries.

## Workflow Summary

```
Edit trade_categories.ltx        (add/remove items from categories)
Edit trader_categories_multipliers.ltx   (adjust min/max per category)
        ↓
Run: python generate_trade_buy.py
        ↓
Copy [trade_generic_buy] from output into trader LTX file
        ↓
Test in game
```

Repeat whenever you want to adjust prices or add new items.

## Tips

- Keep `trade_categories.ltx` as the single source of truth for which items
  belong to which tier. All traders share the same categories file.
- Each trader gets their own multipliers file (or their own section in a shared
  one) since different traders value items differently.
- Items not listed in any category will fall back to the trader's global discount
  rate alone — no per-item override. This is fine for items you haven't
  explicitly categorized yet.
- You can add individual item overrides directly in the trader's LTX file after
  the generated section for one-off exceptions (e.g. a faction patch Sidorovich
  pays a premium for). These will not be overwritten by the tool.
